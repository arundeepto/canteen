# করতোয়া ছাত্রাবাস — মিল ম্যানেজমেন্ট সিস্টেম

Vanilla HTML/CSS/JS + Firebase (Auth + Firestore) দিয়ে তৈরি একটি সম্পূর্ণ মিল ম্যানেজমেন্ট সিস্টেম। কোনো বিল্ড স্টেপ নেই — ফাইলগুলো সরাসরি যেকোনো স্ট্যাটিক হোস্টিং-এ আপলোড করলেই চলবে।

## ফাইল স্ট্রাকচার

```
korotoa-mess/
├── index.html          লগ ইন + বর্ডার নিবন্ধন
├── admin.html           এডমিন ড্যাশবোর্ড
├── boarder.html          বর্ডার ড্যাশবোর্ড
├── css/style.css
├── js/
│   ├── firebase-config.js   ← এখানে তোমার Firebase কনফিগ বসাতে হবে
│   ├── utils.js              মূল্য/তারিখ/টোকেন হেল্পার
│   ├── auth.js (index.html-এর ভেতরেই ইনলাইন)
│   ├── admin.js
│   └── boarder.js
└── firestore.rules       Firestore সিকিউরিটি রুলস
```

## ধাপ ১ — Firebase প্রজেক্ট তৈরি

1. https://console.firebase.google.com এ যাও → **Add project** → নাম দাও (যেমন `korotoa-mess`)।
2. প্রজেক্ট তৈরি হয়ে গেলে বাম মেনু থেকে **Build > Authentication** → **Get started** → **Sign-in method** ট্যাবে গিয়ে **Email/Password** চালু করো।
3. বাম মেনু থেকে **Build > Firestore Database** → **Create database** → **Production mode** সিলেক্ট করো → নিকটতম region দাও (যেমন `asia-south1`)।

## ধাপ ২ — ওয়েব অ্যাপ কনফিগ কপি করা

1. প্রজেক্ট ওভারভিউ পেজে **</>** (Web) আইকনে ক্লিক করে একটি ওয়েব অ্যাপ যোগ করো।
2. যে `firebaseConfig` অবজেক্টটা দেখাবে, সেটা কপি করো।
3. `js/firebase-config.js` ফাইল খুলে `firebaseConfig`-এর মানগুলো বসিয়ে দাও।

## ধাপ ৩ — Firestore সিকিউরিটি রুলস বসানো

1. Firestore Database → **Rules** ট্যাবে যাও।
2. এই প্রজেক্টের `firestore.rules` ফাইলের সম্পূর্ণ কনটেন্ট কপি করে পেস্ট করে **Publish** করো।

## ধাপ ৪ — প্রথম এডমিন অ্যাকাউন্ট তৈরি

নিরাপত্তার জন্য এডমিন অ্যাকাউন্ট রেজিস্ট্রেশন ফর্ম দিয়ে বানানো যায় না — সব রেজিস্ট্রেশন বর্ডার হিসেবেই হয়। ক্যান্টিন মালিকের জন্য:

1. `index.html` থেকে সাধারণ নিবন্ধন ফর্ম দিয়ে একটা অ্যাকাউন্ট খোলো (ব্যাচ/রোল/রুমে যেকোনো মান দিতে পারো, এগুলো এডমিনের জন্য ব্যবহার হবে না)।
2. Firebase Console → Firestore Database → **Data** ট্যাব → `users` কালেকশন → এইমাত্র তৈরি হওয়া ডকুমেন্টটা খুঁজে বের করো (uid অনুযায়ী)।
3. তার `role` ফিল্ডের মান `boarder` থেকে বদলে `admin` করে দাও।
4. এখন এই ইমেইল-পাসওয়ার্ড দিয়ে লগ ইন করলে সরাসরি এডমিন ড্যাশবোর্ডে যাবে।

## ধাপ ৫ — Firestore ইনডেক্স

প্রথমবার ব্যবহারের সময় ব্রাউজার কনসোলে কিছু কুয়েরির জন্য "index required" এরর সহ একটা লিংক আসতে পারে (যেমন বর্ডারের হিসাব/মিল হিস্টোরি লোড করার সময়)। ওই লিংকে ক্লিক করলেই Firebase অটোমেটিক্যালি প্রয়োজনীয় কম্পোজিট ইনডেক্স তৈরি করে দেবে — শুধু **Create index** বাটনে ক্লিক করলেই হবে।

## ধাপ ৬ — হোস্টিং

যেকোনো স্ট্যাটিক হোস্টিং-এ (Firebase Hosting, Netlify, Vercel, GitHub Pages) পুরো ফোল্ডারটা আপলোড করলেই চলবে। Firebase Hosting ব্যবহার করতে চাইলে:

```bash
npm install -g firebase-tools
firebase login
firebase init hosting   # public directory হিসেবে এই ফোল্ডার সিলেক্ট করো
firebase deploy
```

## সিস্টেমের নিয়মকানুন (সংক্ষেপে)

- দুপুরে: ফুল (৳৭০), হাফ (৳৫০)। রাতে: ফুল (৳৭০), হাফ (৳৫০), কোয়ার্টার (৳৩০)।
- প্রতিদিন সকাল ১০টার মধ্যে মিল অন না করলে ওইদিনের জন্য মিল অফ হিসেবে গণ্য হবে (কাটঅফ টাইম `js/utils.js`-এর `MEAL_ON_DEADLINE_HOUR` থেকে বদলানো যায়)।
- হিসাব -২০০ টাকা বা তার কম হয়ে গেলে বর্ডারের ড্যাশবোর্ড লাল হয়ে যাবে এবং নতুন মিল অন করা বন্ধ হয়ে যাবে, যতক্ষণ না এডমিন টাকা জমা নেন (থ্রেশহোল্ড `LOW_BALANCE_THRESHOLD`-এ বদলানো যায়)।
- মিল অন করলেই রঙিন টোকেন (ফুল=সোনালি, হাফ=সবুজ, কোয়ার্টার=লাল) বর্ডারের প্যানেলে দেখা যাবে এবং এডমিনের চেকলিস্টে যোগ হবে।
- এডমিন যেকোনো মিল বাতিল করলে বর্ডারের হিসাবে টাকা স্বয়ংক্রিয়ভাবে ফেরত যাবে।
- সকাল ১০টার পর এডমিন সেদিনের PDF রিপোর্ট ডাউনলোড করতে পারবেন (রিপোর্ট ট্যাব থেকে যেকোনো তারিখের জন্যও দেখা যাবে)।

## একটি গুরুত্বপূর্ণ নোট

এই সিস্টেমে কোনো Cloud Functions ব্যবহার করা হয়নি (যাতে Firebase-এর ফ্রি Spark প্ল্যানেই চালানো যায়)। তাই ব্যালেন্স আপডেট কিছুটা ক্লায়েন্ট-সাইড লজিক + Firestore রুলসের উপর নির্ভরশীল। ছাত্রাবাসের মতো ছোট, পরিচিত গ্রুপের জন্য এটা যথেষ্ট নিরাপদ। ভবিষ্যতে আরও কড়া নিরাপত্তা দরকার হলে ব্যালেন্স আপডেটের লজিকটা একটা Cloud Function-এ সরিয়ে নেওয়া যায়।
